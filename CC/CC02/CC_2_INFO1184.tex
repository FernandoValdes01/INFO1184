\section*{Resumen}

En el presente control se aborda el agrupamiento aglomerativo jerárquico aplicado a un conjunto de datos unidimensional de 10 observaciones, utilizando la norma $L_1$ (Manhattan) como medida de similitud. En la Pregunta 2 se implementa en R la generación de dendogramas, explorando particiones para $k=2$ y $k=3$, y comparando tres métodos de enlace: single, complete y average. En la Pregunta 3 se propone una métrica de separación inter-cluster basada en la distancia entre centroides con norma $L_1$, verificando que cumple las propiedades formales de distancia. Los resultados muestran que complete y average producen agrupamientos más claros para estos datos. Las figuras y el script R se encuentran en el Anexo.

\section{Introducción}

El clustering es una técnica de aprendizaje no supervisado que busca colocar objetos en grupos basados en similaridades, maximizando la similaridad intra-cluster y minimizando la inter-cluster (Lévano, 2026). El agrupamiento aglomerativo jerárquico construye una jerarquía de anidamiento $\Re_0 \subset \Re_1 \subset \cdots \subset \Re_{n-1}$, partiendo de $\Re_0$ donde cada elemento forma su propio grupo, y fusionando progresivamente pares de clusters mediante una función de proximidad $g(R_i, R_j)$ hasta obtener un único conjunto (Duda et al., 2001).

Este trabajo aplica dicho algoritmo al dataset $X = [23.11, -3.33, 1.30, 29.20, -1.57, 34.10, -8.90, -3.40, 9.09, 3.03]$ con norma $L_1$: $d(a,b) = |a - b|$. Se busca: (1) generar dendogramas en R con distintos métodos de enlace, y (2) proponer una métrica inter-cluster con sus propiedades formales.

\subsection{Pregunta 2: Dendogramas en R}

\textbf{a) Script R:} Se calcula la matriz de distancias $L_1$ con \texttt{dist(X, method="manhattan")} y se aplica \texttt{hclust()} que ejecuta el algoritmo aglomerativo: en cada paso encuentra el par $(R_i, R_j)$ tal que $g(R_i, R_j) = \min_{s,r} g(R_r, R_s)$ y genera $\Re_t = (\Re_{t-1} - \{R_i, R_j\}) \cup \{R_q\}$ donde $R_q = R_i \cup R_j$. El script completo y los dendogramas generados se encuentran en el Anexo (secciones A.1 y A.2).

\textbf{b) Particiones $k=2$ y $k=3$:} Usando \texttt{cutree(hc, k=...)} se obtienen las asignaciones (ver Figuras en Anexo A.2). Los agrupamientos cumplen $R_i \neq \varnothing$, $\bigcup R_i = X$, y $R_i \cap R_j = \varnothing$:

\textbf{c) Comparación de métodos:} Los tres métodos difieren en la función $g(R_i, R_j)$: Single usa $d_{min} = \min \|x - x'\|$ (mínima distancia entre pares), Complete usa $d_{max} = \max \|x - x'\|$ (máxima), y Average usa $d_{avg}$ (promedio). Single Linkage produce encadenamiento (chaining) con altura final $\approx 14.02$. Complete Linkage genera clusters más compactos con mayor separación ($43.0$ vs $17.99$). Average ofrece un balance intermedio (altura $29.34$). \textbf{Conclusión:} Complete y Average muestran mejor los agrupamientos naturales, cumpliendo el objetivo de maximizar similaridad intra-cluster y minimizar la inter-cluster. Ver comparación visual en Anexo A.2, Figura~11.

\subsection{Pregunta 3: Métrica de separación inter-cluster ($k=2$)}

Para $k=2$: $R_1 = \{23.11, 29.20, 34.10\}$ y $R_2 = \{-8.90, -3.40, -3.33, -1.57, 1.30, 3.03, 9.09\}$.

\textbf{Métrica propuesta:} Distancia entre centroides con norma $L_1$:
\begin{equation}
D(R_1, R_2) = |c_1 - c_2|, \quad \text{donde } c_i = \frac{1}{|R_i|} \sum_{x \in R_i} x
\end{equation}

\textbf{Cálculo:}
\begin{align}
c_1 &= \frac{23.11 + 29.20 + 34.10}{3} = 28.8033, \qquad
c_2 = \frac{-8.90 + (-3.40) + (-3.33) + (-1.57) + 1.30 + 3.03 + 9.09}{7} = -0.5400 \\
D(R_1, R_2) &= |28.8033 - (-0.5400)| = \boxed{29.3433}
\end{align}

\textbf{Propiedades de la métrica} (según definición de distancia vista en clases, Lévano, 2026):

\begin{enumerate}[leftmargin=1.2cm, itemsep=1pt]
  \item $d(\mathbf{x}_i, \mathbf{x}_j) \geq 0$ \textbf{(No negatividad):} $|c_i - c_j| \geq 0$ por definición del valor absoluto. $\checkmark$
  \item $d(\mathbf{x}_i, \mathbf{x}_i) = 0$ \textbf{(Identidad):} Si $R_i = R_j \Rightarrow c_i = c_j \Rightarrow |c_i - c_j| = 0$. $\checkmark$
  \item $d(\mathbf{x}_i, \mathbf{x}_j) = d(\mathbf{x}_j, \mathbf{x}_i)$ \textbf{(Simetría):} $|c_1 - c_2| = |c_2 - c_1|$. $\checkmark$
  \item $d(\mathbf{x}_i, \mathbf{x}_j) \leq d(\mathbf{x}_i, \mathbf{x}_p) + d(\mathbf{x}_p, \mathbf{x}_j)$ \textbf{(Desigualdad triangular):} Se hereda de la norma $L_1$. $\checkmark$
\end{enumerate}

Adicionalmente, la métrica es \textbf{interpretable} (un valor mayor indica mayor separación) y tiene \textbf{coherencia de escala} (mismas unidades que los datos, consistente con $L_1$). Se compara con alternativas:

\begin{table}[H]
\centering
\footnotesize
\begin{tabular}{lcc}
\toprule
\textbf{Métrica} & \textbf{Fórmula} & \textbf{Valor} \\
\midrule
Distancia entre centroides & $|c_1 - c_2|$ & 29.3433 \\
$d_{min}$ (single) & $\min |x - x'|$ & 14.02 \\
$d_{max}$ (complete) & $\max |x - x'|$ & 43.00 \\

Se elige la distancia entre centroides porque: resume cada $R_i$ en un punto representativo, es robusta ante extremos, eficiente $O(n)$ y consistente con la norma $L_1$.

\section{Reflexión}

Este control permitió aplicar el algoritmo aglomerativo jerárquico siguiendo el procedimiento de clases: desde $\Re_0$ con $n$ grupos unitarios, fusionando en cada paso el par que minimiza $g(R_i, R_j)$. Se observó que Single Linkage ($d_{min}$) produce chaining, mientras Complete ($d_{max}$) y Average ($d_{avg}$) generan clusters más compactos. En la Pregunta 3, verificar las cuatro propiedades de distancia para la métrica propuesta reforzó la formalización vista en clases. La implementación en R complementó el entendimiento teórico con verificación computacional.

\subsection*{A.1 Script R completo (Pregunta 2)}

\begin{lstlisting}
# ============================================================
# Control 2 - INFO1184 - Pregunta 2
# Agrupamiento Aglomerativo Jerarquico con R
# ============================================================

# --- Datos ---
X <- c(23.11, -3.33, 1.30, 29.20, -1.57, 34.10,
       -8.90, -3.40, 9.09, 3.03)
names(X) <- paste0("x", 1:10)
labels_val <- paste0("x", 1:10, "(", X, ")")

# Matriz de distancias L1 (Manhattan)
d <- dist(X, method = "manhattan")

# --- a) Dendograma basico ---
hc_single <- hclust(d, method = "single")
plot(hc_single, labels = labels_val,
     main = "Dendograma - Single Linkage (L1)",
     xlab = "Datos", ylab = "Distancia L1", hang = -1)

# --- b) Particiones k=2 y k=3 ---
plot(hc_single, labels = labels_val,
     main = "Single Linkage - Particion k=2",
     xlab = "Datos", ylab = "Distancia L1", hang = -1)
rect.hclust(hc_single, k = 2, border = c("red","blue"))
clusters_k2 <- cutree(hc_single, k = 2)

plot(hc_single, labels = labels_val,
     main = "Single Linkage - Particion k=3",
     xlab = "Datos", ylab = "Distancia L1", hang = -1)
rect.hclust(hc_single, k = 3,
            border = c("red","blue","green"))
clusters_k3 <- cutree(hc_single, k = 3)

# --- c) Tres metodos ---
hc_complete <- hclust(d, method = "complete")
hc_average  <- hclust(d, method = "average")

par(mfrow = c(1, 3))
plot(hc_single, labels = labels_val,
     main = "Single Linkage", hang = -1, cex = 0.7)
rect.hclust(hc_single, k=2, border=c("red","blue"))

plot(hc_complete, labels = labels_val,
     main = "Complete Linkage", hang = -1, cex = 0.7)
rect.hclust(hc_complete, k=2, border=c("red","blue"))

plot(hc_average, labels = labels_val,
     main = "Average Linkage", hang = -1, cex = 0.7)
rect.hclust(hc_average, k=2, border=c("red","blue"))
\end{lstlisting}
